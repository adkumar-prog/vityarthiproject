### Key Commands

| Choice | Action | Notes |
| :---: | :--- | :--- |
| **1** | **Add Task** | Prompts for description and priority (low, medium, high). |
| **2** | **View Tasks** | Lists Pending tasks (numbered and sorted by priority) and Completed tasks. |
| **3** | **Complete Task**| Requires you to enter the **number** corresponding to the task in the **Pending** list. |
| **4** | **Exit** | Saves all current tasks to `tasks.pkl` and closes the application. |

## ‚öôÔ∏è Project Structure

The application is built using two main classes:

* **`Task`**: Represents a single item with a description, priority, creation date, and completion status.
* **`TaskManager`**: Manages a list of `Task` objects, handling operations like adding, viewing, completing, and exiting.
